✘Not implementedartifact-bundle

The artifact-bundle resource defines which artifacts (for example, traits, object types, roles, etc.) are contained within the artifact-bundle.

Media type

The media type for this resource is .

PropertyDescription
nameHuman readable identification
versionA semantic version number
require-bundleObject ids of other artifact bundles that this artifact bundle depends on
symbolic-nameThe namespace of the artifact bundle (think Maven groupId). The name attribute will have the actual artifact bundle name (think Maven artifactId)
bundle-install-statusThe overall installed status of the artifact bundle. If any artifact-install-status are failed then this will be failed too
Schema for GET
<xs:element name="artifact-bundle">
  <xs:complexType>
    <xs:complexContent>
      <xs:extension> base="versioned">
        <xs:sequence>
          <xs:element name="require-bundle" type="xs:string"/>
          <xs:element name="symbolic-name" type="xs:string"/>
          <xs:element name="bundle-install-status" type="xs:string"/>  
          <xs:element ref="resource" minOccurs="0" maxOccurs="unbounded"/>
        </xs:sequence>
      </xs:extension>
    </xs:complexContent>
  </xs:complexType>
</xs:element>

<xs:complexType name="versioned">
  <xs:sequence>
    <xs:element name="name" type="qualified-name"/>
    <xs:element name="version" type="version"/>
  </xs:sequence>
</xs:complexType>

<xs:simpleType name="qualified-name">
  <xs:restriction base="xs:string">
    <xs:pattern value="[a-z][a-z0-9_]+(\.[a-zA-Z0-9_]+)*"/>
  </xs:restriction>
</xs:simpleType>

<xs:simpleType name="version">
  <xs:restriction base="xs:string">
    <xs:pattern value="\d+(\.\d+){0,3}(\-[a-zA-Z0-9]+)?"/>
  </xs:restriction>
</xs:simpleType>

<xs:element name="resource">
  <xs:complexType>
    <xs:choice>
      <xs:element ref="link"/>
      <xs:element ref="template"/>
    </xs:choice>
    <xs:attribute name="rel" type="xs:anyURI" use="required"/>
  </xs:complexType>
</xs:element>

<xs:element name="link">
  <xs:complexType>
    <xs:attribute name="href" type="xs:anyURI" use="required"/>
  </xs:complexType>
</xs:element>

<xs:element name="template">
  <xs:complexType>
    <xs:sequence maxOccurs="unbounded" minOccurs="0">
      <xs:element name="var">
        <xs:complexType>
          <xs:attribute name="name" use="required"/>
          <xs:attribute name="uri" type="xs:anyURI" use="required"/>
        </xs:complexType>
      </xs:element>
    </xs:sequence>
    <xs:attribute name="href-template" use="required"/>
  </xs:complexType>
</xs:element>

Media type

The media type for this resource is .

PropertyDescription
symbolic-nameThe symbolic name of the OSGi bundle
versionThe version of the OSGi bundle
file-nameThe name of the jar file representing the OSGi bundle on the file system

✔Fully implementedtenant-by-name

The tenant-by-name resource allows one to retrieve details about a tenant without knowing its ID. The response to a GET on this resource will be a redirect (303 See Other) to the tenant with the supplied name (or 404 Not Found when no tenant with the given name could be found). The response will include an x-ngis-tenant-id header containing the tenant's ID.
